# AIHR 文章 GEO 优化写作清单（发布前对照）

> **GEO = Generative Engine Optimization（生成式引擎优化）**
> 目标：让文章在被 AI 搜索引擎 / LLM（ChatGPT、Perplexity、豆包、元宝、Copilot）生成答案时被引用。
> 适用：每篇新文章撰写完成、发布前逐条对照。SEO 管「人搜得到」，GEO 管「AI 答得出」——两者互补，不可互相替代。

---

## 一、内容结构（让 LLM 愿意引）

- [ ] **首段 2–3 句直给答案（Answer-first）**
  开头就给结论 / 定义 / 关键数字，不铺垫。LLM 摘要优先取首段。
  ✅ 「字节 2026 春招 7000 名实习生，转正率超 50%，本质是用高潜替代 8 年熟练工。」
  ❌ 「在 AI 浪潮下，组织正在发生深刻变革……」

- [ ] **每段一个可引用结论 + 硬支撑**
  每段必须有数据 / 案例 / 引用，拒绝「趋势」「赋能」「闭环」等空词。
  ✅ 「大厂降低社招经验权重——阿里、腾讯、美团同期调整。」

- [ ] **小标题用陈述句或问题（匹配搜索意图）**
  ✅ 「字节为什么用 7000 实习生替代老员工？」
  ❌ 「背景介绍」「相关分析」

- [ ] **FAQ 段落（问答对）**
  用真实用户会搜的问题作小标题，LLM 常直接抽取 FAQ 作答。每篇至少 3 条。

- [ ] **列表 / 表格优先**
  步骤、对比、要点用有序 / 无序列表；数据对比用表格——LLM 抽取成功率远高于长段落。

- [ ] **三类高引内容必含其一**
  定义（这是什么）、对比（A vs B 差异）、步骤（怎么做）。

## 二、实体与表述（让 LLM 引得准）

- [ ] **实体首次出现写全称 + 缩写**
  「DRI（Directly Responsible Individual，直接责任人）」「AIHR（AI + HR）」。
  降低 LLM 实体消歧错误，提升被准确引用的概率。

- [ ] **避免模糊代词与营销腔**
  「其」「该」「相关方面」→ 写具体主语；禁用「震惊」「重磅」「颠覆」「绝绝子」。

- [ ] **关键数字显性化**
  把「大量」「显著提升」改成「3500 人」「+50%」。LLM 偏好可核实的具体值。

## 三、技术层（让 LLM 抓得到）

- [ ] **结构化数据合规**：Article + FAQPage + BreadcrumbList 已验证合法（跑 `tools/publish_check.py`）。
- [ ] **meta description 从首段提取**，含核心结论与实体。
- [ ] **og:image 指向 `/assets/og-covers/og-{slug}.jpg`**。
- [ ] **标题 ≤ 28 字，含核心实体 / 关键词**（百度、必应标题偏好）。
- [ ] **内部链接（相关阅读 3–5 篇）**：帮 LLM 与爬虫理解主题簇，强化「AI+HR 组织」权威。
- [ ] **二维码铁律**：每篇仅 1 个 `article-footer-qr`，无残留 `article-qrcode` / footer QR。

## 四、发布前最后一步

